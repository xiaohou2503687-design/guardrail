# 🛡️ GuardRail — 独立开发者安全扫描器

> **一句话**: `npx guardrail scan` — 5秒扫描你的项目，找出密钥泄露、OWASP漏洞、依赖风险。

---

## 📊 立体评估

### 1. 市场维度 ⭐⭐⭐⭐ (4/5)

| 指标 | 数据 |
|------|------|
| **TAM** | 全球 DevSecOps 市场 $15B+ (2026) |
| **SAM** | 独立开发者 + 小型团队 ~5M |
| **SOM** | 第一年 3K-15K 用户 |
| **竞争格局** | Snyk(企业级)、GitHub Dependabot(免费但慢)、Trivy(容器扫描) |
| **市场缺口** | 没有面向独立开发者的"一键安全体检"CLI 工具 |

**关键信号:**
- 每天都有开发者不小心把 API Key 提交到 GitHub
- GitHub Secret Scanning 只在 push 后提醒，不是事前预防
- 独立开发者 90% 没有任何安全扫描
- "我被入侵了"在 r/webdev 和 Twitter 上月均 50+ 帖

### 2. 技术维度 ⭐⭐⭐⭐ (4/5)

| 维度 | 评估 |
|------|------|
| **复杂度** | 中低 — 规则引擎 + 扫描器 |
| **核心依赖** | 静态分析(自定义规则)、依赖检查(npm audit封装)、密钥检测(gitleaks) |
| **数据存储** | 无/本地报告文件 |
| **AI 层** | 误报过滤 + 修复建议生成 |
| **估时** | 1.5周 MVP，3周完整版 |
| **技术风险** | ✅ 低 — 主要是规则集合，不涉及复杂系统 |

### 3. 营收维度 ⭐⭐⭐⭐⭐ (5/5)
